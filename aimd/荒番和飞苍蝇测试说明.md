# 荒番和飞苍蝇功能测试说明

## 📋 功能概述

### 1. 荒番规则
- **触发条件**：流局（牌墙摸完，四家都没胡）
- **效果**：荒番数 +1，下一局胡牌时总分 ×2ⁿ
- **显示**：结算界面显示荒番数和翻倍倍数

### 2. 飞苍蝇规则
- **触发条件**：胡牌后（有牌型：门清、混一色、清一色、七对、大单吊、碰碰胡）
- **翻牌位置**：从牌墙尾部随机翻一张
- **苍蝇分值**：
  - 数字牌 1：+10 分
  - 数字牌 2-9：牌面值分（如 6=6 分）
  - 风牌/白发中：+5 分
  - 花牌：+5 分
- **显示**：结算界面显示苍蝇牌和苍蝇分

## 🧪 测试步骤

### 测试 1：基本胡牌（无苍蝇）
1. 启动服务器
2. 创建房间并开始游戏
3. 正常胡牌（基本胡，无特殊牌型）
4. **预期结果**：
   - 结算界面显示"基本胡 0 番"
   - 不显示苍蝇分区域（因为没有 qualifying 牌型）
   - 不显示荒番区域（如果不是荒番局）

### 测试 2：门清胡牌 + 飞苍蝇
1. 不碰不杠，门清胡牌
2. **预期结果**：
   - 结算界面显示"门清 1 番"
   - 显示苍蝇分区域
   - 显示苍蝇牌（从牌墙尾部翻的牌）
   - 苍蝇分根据翻到的牌计算

### 测试 3：流局 + 荒番
1. 故意让牌墙摸完，无人胡牌
2. **预期结果**：
   - 结算界面显示"流局"
   - 显示荒番信息："荒番数 1"
   - 说明："下一局胡牌 ×2 倍"

### 测试 4：荒番局胡牌
1. 先制造一次流局（荒番 1）
2. 下一局胡牌（门清）
3. **预期结果**：
   - 结算界面显示"门清 1 番"
   - 显示苍蝇分（如果有）
   - 显示荒番信息："荒番 1 (×2)"
   - 最终得分 = (花数×2^番数 + 苍蝇分) × 2

### 测试 5：多次荒番叠加
1. 连续制造 2 次流局（荒番 2）
2. 第 3 局胡牌
3. **预期结果**：
   - 荒番信息显示："荒番 2 (×4)"
   - 最终得分 = (花数×2^番数 + 苍蝇分) × 4

### 测试 6：大单吊 + 苍蝇
1. 碰/杠 4 组，最后单钓一对将胡牌
2. **预期结果**：
   - 结算界面显示"大单吊 1 番"
   - 显示苍蝇分区域
   - 苍蝇分根据翻到的牌计算

### 测试 7：七对子 + 苍蝇
1. 凑成 7 对子胡牌
2. **预期结果**：
   - 结算界面显示"七对子 2 番"
   - 显示苍蝇分区域
   - 苍蝇分根据翻到的牌计算

## 🔍 检查点

### 服务器日志
启动服务器后，观察以下日志：
```
本局为荒番局，荒番数：X
胡牌后飞苍蝇：{"type":"...","value":...}
流局！荒番数 +1，当前荒番数：X
```

### 客户端显示
结算界面应显示：
1. ✅ 番数明细（包括门清、大单吊等）
2. ✅ 花数明细（底花 + 花牌 + 杠/刻子）
3. ✅ 苍蝇分（如果有 qualifying 牌型）
   - 苍蝇牌显示
   - 苍蝇分值
4. ✅ 荒番信息（如果是荒番局）
   - 荒番数
   - 翻倍倍数

### 计分验证
计分公式：
```
最终得分 = (花数 × 2^番数 + 苍蝇分) × 荒番倍数
```

示例：
- 门清（1 番）+ 2 花 + 苍蝇 5 分 + 荒番 1（×2）
- 得分 = (2 × 2^1 + 5) × 2 = (4 + 5) × 2 = 18 分

## 🐛 已知问题排查

### 问题 1：不显示苍蝇分
**原因**：没有 qualifying 牌型
**解决**：确保胡牌时有门清、混一色、清一色、七对、大单吊、碰碰胡等牌型

### 问题 2：不显示荒番信息
**原因**：没有流局过
**解决**：先制造一次流局（让牌墙摸完）

### 问题 3：苍蝇分计算错误
**检查**：
1. 服务器日志中的苍蝇牌信息
2. 客户端显示的苍蝇牌
3. 苍蝇分值计算逻辑

### 问题 4：荒番翻倍不正确
**检查**：
1. `roundResult.scoreResult.huangFanMultiplier` 是否正确
2. 计分公式是否正确应用翻倍
