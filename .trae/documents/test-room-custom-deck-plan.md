# TEST房间自定义牌谱功能实施计划

## 功能概述
在现有上海麻将游戏基础上，增加一个特殊的测试房间（房间号为 `TEST`），进入该房间后在准备阶段可以自定义牌谱顺序，游戏开始后按照自定义的牌谱顺序发牌。

## 一、服务端修改 (server.js)

### 1.1 MahjongRoom 类添加测试房间属性
**位置**: `server.js:204-235` (MahjongRoom 构造函数)

添加属性：
- `isTestRoom`: 布尔值，标识是否为测试房间
- `customDeckOrder`: 数组，存储自定义的牌谱顺序

```javascript
// 在构造函数中添加
this.isTestRoom = code === 'TEST';
this.customDeckOrder = null;  // 自定义牌谱顺序
```

### 1.2 修改 createDeck 函数支持自定义顺序
**位置**: `server.js:103-131`

创建新函数 `createDeckWithOrder(order)`:
- 接收牌谱顺序数组作为参数
- 按照指定顺序创建牌组
- 如果没有指定顺序，使用默认随机洗牌

### 1.3 修改 startGame 函数
**位置**: `server.js:698-724`

修改发牌逻辑：
- 检查是否为测试房间且有自定义牌谱
- 如果是，使用 `createDeckWithOrder` 创建有序牌组
- 否则使用原有的随机洗牌逻辑

### 1.4 添加 Socket 事件处理
**位置**: `server.js:3375` 附近

新增事件：
- `set_custom_deck`: 接收客户端发送的自定义牌谱
- 验证牌谱有效性（包含所有必要的牌）
- 存储到 room.customDeckOrder

### 1.5 修改 join_room 事件处理
**位置**: `server.js:3306-3373`

- 允许房间号为 `TEST` 的房间创建
- `TEST` 房间不使用随机生成的房间码

## 二、客户端修改 (index.html)

### 2.1 添加牌谱编辑器 UI
**位置**: 在 `roomScreen` 内，准备按钮上方

创建牌谱编辑器容器：
- 仅在 TEST 房间显示
- 显示所有麻将牌的可拖动列表
- 提供重置为默认顺序的按钮
- 提供保存牌谱的按钮

### 2.2 牌谱编辑器样式
**位置**: CSS 部分

添加样式：
- `.deck-editor-container`: 编辑器容器
- `.deck-tile-item`: 单张牌样式
- `.deck-tile-item.dragging`: 拖动中的样式
- `.deck-category`: 牌类型分组（万、条、筒、风、箭）

### 2.3 牌谱编辑器 JavaScript 逻辑

#### 2.3.1 初始化牌谱
创建函数 `initDefaultDeckOrder()`:
- 生成默认牌谱顺序：
  - 4张一万、4张二万...4张九万
  - 4张一条、4张二条...4张九条
  - 4张一筒、4张二筒...4张九筒
  - 4张中、4张发、4张白（箭牌）
  - 4张东、4张南、4张西、4张北（风牌）
- 花牌放在最后（8张）

#### 2.3.2 渲染牌谱编辑器
创建函数 `renderDeckEditor()`:
- 渲染可拖动的牌列表
- 每张牌显示麻将图案
- 支持拖拽排序

#### 2.3.3 拖拽功能实现
使用 HTML5 Drag and Drop API:
- `dragstart` 事件：记录被拖动的牌
- `dragover` 事件：显示可放置位置
- `drop` 事件：交换牌的位置

#### 2.3.4 保存牌谱
创建函数 `saveCustomDeck()`:
- 收集当前牌谱顺序
- 通过 Socket 发送到服务端
- 显示保存成功提示

### 2.4 修改房间加入逻辑
**位置**: `joinRoom()` 函数

- 检测是否加入 TEST 房间
- 如果是 TEST 房间，显示牌谱编辑器
- 普通房间隐藏牌谱编辑器

### 2.5 修改房间创建逻辑
**位置**: `createRoom()` 函数

- 检测是否创建 TEST 房间
- TEST 房间直接使用固定房间号

## 三、数据结构设计

### 3.1 牌谱顺序数组格式
```javascript
// 每个元素代表一张牌，按发牌顺序排列
// 索引0是牌堆顶部（最后发），索引末尾是牌堆底部（最先发）
customDeckOrder = [
    { type: 'wan', value: 1, id: 'wan_1_0' },
    { type: 'wan', value: 1, id: 'wan_1_1' },
    // ... 其他牌
]
```

### 3.2 默认牌谱顺序
```
[一万x4, 二万x4, ..., 九万x4,
 一条x4, 二条x4, ..., 九条x4,
 一筒x4, 二筒x4, ..., 九筒x4,
 中x4, 发x4, 白x4,
 东x4, 南x4, 西x4, 北x4,
 春, 夏, 秋, 冬, 梅, 兰, 竹, 菊]
```

## 五、注意事项

1. **牌谱验证**: 服务端需要验证牌谱包含所有144张牌
2. **花牌处理**: 花牌在发牌时自动补花，需要确保花牌在牌谱中
3. **安全性**: 只允许 TEST 房间使用自定义牌谱
4. **UI 响应**: 拖拽操作需要流畅，提供视觉反馈
5. **兼容性**: 不影响普通房间的正常游戏流程
