# 副露区UI修改计划 - 横牌指向放牌人

## 需求概述
修改副露区（吃、碰、杠）的UI显示，使横放的那张牌的朝向指向出牌玩家（放牌人），符合上海敲麻/国标通用桌面摆放规范。

## 座位关系说明
- **本家**（假设坐东位，seatIndex = 0）
- **上家**（北，seatIndex = 3）：只能从上家吃牌
- **下家**（南，seatIndex = 1）
- **对家**（西，seatIndex = 2）

## 横牌朝向规则
| 副露类型 | 来源 | 横牌位置 | 朝向 |
|---------|------|---------|------|
| 吃上家 | 上家(北) | 左侧 | 朝左 ← |
| 碰上家 | 上家(北) | 左侧 | 朝左 ← |
| 碰对家 | 对家(西) | 中间 | 朝前/朝上 ↑ |
| 碰下家 | 下家(南) | 右侧 | 朝右 → |

## 技术实现方案

### 1. 添加CSS样式
- 添加 `.meld-tile-rotated` 类用于横牌旋转
- 添加 `.meld-tile-left` (朝左)、`.meld-tile-center` (朝前)、`.meld-tile-right` (朝右) 样式

### 2. 创建方向计算函数
创建 `getMeldTileDirection(meld, mySeatIndex)` 函数：
- 输入：meld对象（含from字段）、mySeatIndex
- 输出：'left' | 'center' | 'right' | 'none'（暗杠无横牌）

### 3. 修改前端渲染函数

#### 3.1 修改 `renderMyMelds` 函数（约5023行）
- 遍历每个meld
- 根据 `meld.from` 与 `mySeatIndex` 的关系确定横牌位置
- 为横牌添加对应的CSS类

#### 3.2 修改对手副露渲染（约4821行）
- 在 `updateGameUI` 中修改对手副露渲染逻辑
- 同样根据 `from` 字段确定横牌方向

### 4. 数据结构确认
服务器已在 `player.melds` 中存储了 `from` 字段：
- `from: lastDiscardPlayer` - 吃、碰、杠的来源玩家索引
- `from: player.seatIndex` - 暗杠的from是自己

## 具体代码修改

### 步骤1: 添加CSS样式
```css
.meld-group .tile.rotated-left { transform: rotate(-90deg); }
.meld-group .tile.rotated-center { transform: rotate(0deg); }
.meld-group .tile.rotated-right { transform: rotate(90deg); }
```

### 步骤2: 添加方向计算函数
```javascript
function getMeldTileDirection(meld, mySeatIndex) {
    if (!meld.from && meld.from !== 0) return 'none';
    if (meld.type === 'an_gang' || meld.type === 'gang' && meld.from === mySeatIndex) return 'none';
    
    const fromSeat = meld.from;
    const diff = (fromSeat - mySeatIndex + 4) % 4;
    
    // diff: 1=下家(右), 2=对家(中), 3=上家(左)
    if (diff === 3 || diff === -1 || diff === -1) return 'left';  // 上家
    if (diff === 2) return 'center';  // 对家
    if (diff === 1) return 'right';  // 下家
    
    return 'none';
}
```

### 步骤3: 修改渲染函数
- 在渲染meld时，为对应位置的牌添加旋转样式
- 吃牌：横牌在左侧（index 0）
- 碰牌：横牌根据来源在左/中/右侧
- 杠牌：同上
