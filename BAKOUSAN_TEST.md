# 包三口规则测试指南

## 功能说明

包三口是上海麻将的特殊规则，当玩家之间通过吃/碰/明杠累计形成 3 次关系时触发特殊赔付规则。

## 成立条件

- 玩家 A 与玩家 B 通过吃/碰/明杠累计形成 3 次关系
- 每次吃、碰、明杠均计 1 口
- **注意**：暗杠不计入口数（因为是摸牌，不是吃碰杠）

## 赔付规则

### 1. 自摸胡牌
- **正常情况**：三家各付 1 份（共 3 份）
- **包三口情况**：若胡牌者 A 与玩家 B 有三口关系，玩家 B 单独付 5 份

### 2. 点炮胡牌
- **正常情况**：放炮者付 1 份
- **包三口情况**：若胡牌者 A 与放炮者 B 有三口关系，放炮者 B 付 2 份

## 测试场景

### 场景 1：自摸包三口
1. 玩家 A 碰了玩家 B 的牌 3 次（或吃 2 次 + 碰 1 次等组合）
2. 玩家 A 自摸胡牌
3. **预期结果**：
   - 玩家 B 单独赔付 5 份给玩家 A
   - 其他玩家 C、D 不赔付
   - 结算界面显示"自摸包三口"，显示累计口数（如 3 口）

### 场景 2：点炮包三口
1. 玩家 A 碰了玩家 B 的牌 3 次
2. 玩家 B 打出的牌被玩家 A 点炮胡牌
3. **预期结果**：
   - 玩家 B 赔付 2 份给玩家 A
   - 结算界面显示"点炮包三口"，显示累计口数（如 3 口）

### 场景 3：无三口关系
1. 玩家 A 与玩家 B 只有 2 次或以下的吃碰杠
2. 玩家 A 自摸或点炮胡牌
3. **预期结果**：
   - 按正常规则赔付（自摸 3 份，点炮 1 份）
   - 不显示包三口信息

### 场景 4：暗杠不计数
1. 玩家 A 暗杠（自己摸到 4 张相同的牌）
2. 玩家 A 碰了玩家 B 的牌 2 次
3. 玩家 A 自摸胡牌
4. **预期结果**：
   - 暗杠不计入口数，累计只有 2 口
   - 不触发包三口规则
   - 按正常自摸赔付

## 测试步骤

### 方法 1：单机测试（solo 模式）
1. 打开 `solo/index.html`
2. 开始游戏
3. 通过控制台或修改代码强制触发碰/吃操作
4. 观察日志中的三口计数输出

### 方法 2：多人联机测试
1. 启动服务器：`node server.js`
2. 打开浏览器访问 `http://localhost:3000`
3. 创建房间，邀请好友或使用 AI 填充
4. 进行游戏，故意制造吃碰杠场景
5. 胡牌后查看结算界面的包三口信息

## 代码日志

游戏运行时，服务器控制台会输出三口计数日志：
```
【三口】玩家 A 碰了 玩家 B 的牌，累计 1 口
【三口】玩家 A 吃了 玩家 B 的牌，累计 2 口
【三口】玩家 A 明杠了 玩家 B 的牌，累计 3 口
【包三口】自摸：玩家 B 包 玩家 A 的三口，累计 3 口
```

## 界面显示

结算时，如果触发包三口规则，会在"荒番"信息下方显示：

```
🎯 包三口
━━━━━━━━━━━━━━━
自摸包三口    3 口
玩家 B 与 玩家 A 有三口关系，单独赔付 5 份
```

## 注意事项

1. 包三口只计算吃、碰、明杠，不包括暗杠
2. 包三口关系是单向的（A 吃 B 的牌，只计 A 对 B 的口数）
3. 自摸时只检查胡牌者对其他玩家的口数
4. 点炮时只检查胡牌者对放炮者的口数
5. 包三口赔付会覆盖正常赔付规则
