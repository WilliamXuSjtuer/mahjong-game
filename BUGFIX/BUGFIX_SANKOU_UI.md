# Bug 修复：包三口 UI 提示逻辑错误

## 修复日期
2026-03-02

## 问题描述

玩家已经与其他玩家形成包三口关系（累计吃/碰/明杠 3 次），但头像上没有显示红色包三口指示器徽章。

### 具体场景（参考截图）
- 玩家 642（西家）已经碰了 3 组牌
- 假设这 3 组牌都是吃的/碰的同一位玩家的牌
- **预期**：该玩家的头像上应该显示红色包三口徽章（数字"3"）
- **实际**：没有任何包三口提示

---

## 根本原因

在 `updateSankouIndicators()` 函数中，包三口计数的检查逻辑错误：

### 错误代码
```javascript
// ❌ 错误逻辑
// 获取该玩家对当前玩家的三口计数
const sankouCount = player.sankouCounts ? player.sankouCounts[mySeatIndex] : 0;
```

### 问题分析
- 原代码检查的是：**其他玩家对当前玩家的三口计数**
- 但包三口规则应该是：**当前玩家对其他玩家的三口计数**

### `sankouCounts` 的正确含义
```javascript
player.sankouCounts[X] = N
// 表示：player 对 X 玩家有 N 口的包三口关系
// 即：player 吃了/碰了/明杠了 X 玩家的牌 N 次
```

---

## 修复方案

### 修复后的代码
```javascript
// ✅ 正确逻辑
// 【修复】获取当前玩家对该玩家的三口计数
// sankouCounts 的含义：player.sankouCounts[X] 表示 player 对 X 玩家的三口计数
const sankouCount = currentPlayer.sankouCounts ? currentPlayer.sankouCounts[i] : 0;
```

### 修复说明
- 使用 `currentPlayer.sankouCounts[i]` 代替 `player.sankouCounts[mySeatIndex]`
- 检查**当前玩家对其他玩家的三口计数**
- 当计数 ≥ 3 时，在该玩家的头像上显示红色徽章

## 包三口规则回顾

### 成立条件
- 玩家 A 与玩家 B 通过吃/碰/明杠累计形成 3 次关系
- 每次吃、碰、明杠均计 1 口
- **注意**：暗杠不计入口数（因为是摸牌）

### 赔付规则
| 场景 | 正常赔付 | 包三口赔付 |
|------|---------|-----------|
| 自摸 | 三家各 1 份（共 3 份） | 包三口者单独 5 份 |
| 点炮 | 放炮者 1 份 | 放炮者 2 份 |

## 技术说明

### 数据结构
```javascript
// 玩家对象
{
    sankouCounts: [0, 0, 0, 0]  // 索引 0-3 对应四个玩家
}

// sankouCounts[0] = 3 表示：该玩家对玩家 0 有 3 口的包三口关系
```

### UI 更新流程
1. 游戏状态更新时调用 `_doUpdateGameUI()`
2. `_doUpdateGameUI()` 调用 `updateSankouIndicators()`
3. `updateSankouIndicators()` 遍历所有玩家
4. 对每个玩家，检查当前玩家对其的三口计数
5. 如果计数 ≥ 3，显示红色徽章；否则隐藏

### 视觉效果
- **位置**：头像右上角
- **样式**：红色圆形徽章，白色边框
- **动画**：脉冲效果（2 秒周期）
- **内容**：显示累计口数（如"3"、"4"）
