# Bug 修复：包三口计数器每局重置

## 修复日期
2026-03-02

## 问题描述

包三口计数器没有在每小局结束时重置，导致：
- 第一局形成的包三口关系会延续到第二局
- 第二局又继续累加，导致计数错误
- 可能出现第二局开始就有包三口关系的错误情况

### 具体场景
1. 第一局：玩家 A 碰了玩家 B 的牌 2 次
2. 第二局：玩家 A 又碰了玩家 B 的牌 1 次
3. **错误结果**：累计 3 口，触发包三口规则
4. **正确结果**：第二局应该重新开始计数，只有 1 口

---

## 根本原因

在 `startGame()` 函数中，每局开始时会重置玩家的大部分状态（手牌、副露、弃牌、花牌、听牌状态等），但**没有重置包三口计数器**。

### 原始代码（第 698-714 行）
```javascript
// 发牌：每人 13 张，庄家 14 张（花牌自动补花）
this.players.forEach((player, index) => {
    player.hand = [];
    player.melds = [];
    player.discards = [];
    player.flowers = [];
    player.isTing = false;
    player.isQiao = false;
    // ❌ 缺少：player.sankouCounts = [0, 0, 0, 0];
    
    // ...
});
```

---

## 修复方案

在 `startGame()` 函数中，每局开始时重置所有玩家的包三口计数器：

### 修复后的代码
```javascript
// 发牌：每人 13 张，庄家 14 张（花牌自动补花）
this.players.forEach((player, index) => {
    player.hand = [];
    player.melds = [];
    player.discards = [];
    player.flowers = [];
    player.isTing = false;
    player.isQiao = false;
    player.sankouCounts = [0, 0, 0, 0];  // ✅ 【新增】每局重置包三口计数器
    
    // ...
});
```

## 包三口规则说明

### 计数规则
- **范围**：每小局独立计数，跨局不累加
- **计入口数**：吃、碰、明杠（每次计 1 口）
- **不计入**：暗杠（因为是摸牌，不是吃碰杠）

### 赔付规则
| 场景 | 正常赔付 | 包三口赔付 |
|------|---------|-----------|
| 自摸 | 三家各 1 份（共 3 份） | 包三口者单独 5 份 |
| 点炮 | 放炮者 1 份 | 放炮者 2 份 |

## 技术说明

### 数据结构
```javascript
player.sankouCounts = [0, 0, 0, 0]
// 索引 0-3 对应四个玩家（按座位索引）
// sankouCounts[X] 表示该玩家对 X 玩家的三口计数
```

### 重置时机
- **时机**：每小局开始时（`startGame()` 函数）
- **位置**：在重置玩家手牌、副露等状态时同时重置
- **原因**：包三口是**本局内的临时关系**，跨局无效

### 为什么需要每局重置？
1. **规则要求**：包三口是局内关系，每局重新建立
2. **公平性**：避免上局的包三口关系影响下局
3. **逻辑清晰**：每局都是新的开始，计数清晰
