# 数据结构文档

本文档详细描述项目中使用的所有核心数据结构。

## 一、麻将牌数据结构

### 1.1 牌的基本结构 (Tile)

```typescript
interface Tile {
    type: 'wan' | 'tiao' | 'tong' | 'wind' | 'honor' | 'flower';
    value: number | string;
    id: string;
}
```

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| type | string | 牌的类型 | 'wan', 'tiao', 'tong', 'wind', 'honor', 'flower' |
| value | number/string | 牌的值 | 数字牌: 1-9, 风牌: 'dong','nan','xi','bei', 箭牌: 'zhong','fa','bai' |
| id | string | 唯一标识符 | 'wan_5_0', 'wind_dong_1', 'flower_chun' |

### 1.2 牌的类型常量

```javascript
// 数字牌类型
const TILE_TYPES = ['wan', 'tiao', 'tong'];  // 万、条、筒

// 数字牌值
const TILE_VALUES = [1, 2, 3, 4, 5, 6, 7, 8, 9];

// 风牌
const WINDS = ['dong', 'nan', 'xi', 'bei'];   // 东南西北
const WIND_NAMES = { dong: '东', nan: '南', xi: '西', bei: '北' };

// 箭牌（中发白）
const HONORS = ['zhong', 'fa', 'bai'];

// 花牌
const FLOWERS = ['chun', 'xia', 'qiu', 'dong_hua', 'mei', 'lan', 'zhu', 'ju'];
const FLOWER_NAMES = {
    chun: '春', xia: '夏', qiu: '秋', dong_hua: '冬',
    mei: '梅', lan: '兰', zhu: '竹', ju: '菊'
};
```

### 1.3 牌组构成

| 类型 | 数量 | 说明 |
|------|------|------|
| 万子 | 36张 (9种×4) | 一万到九万 |
| 条子 | 36张 (9种×4) | 一条到九条 |
| 筒子 | 36张 (9种×4) | 一筒到九筒 |
| 风牌 | 16张 (4种×4) | 东南西北 |
| 箭牌 | 12张 (3种×4) | 中发白 |
| 花牌 | 8张 (8种×1) | 春夏秋冬梅兰竹菊 |
| **总计** | **144张** | - |

## 二、玩家数据结构

### 2.1 玩家对象 (Player)

```typescript
interface Player {
    id: string;              // Socket ID 或 AI ID
    username: string;        // 玩家名称
    avatar: string;          // 头像 emoji
    voice: string;           // 语音类型
    socket: Socket | null;   // Socket 连接 (AI为null)
    ready: boolean;          // 是否准备
    seatIndex: number;       // 座位索引 (0-3)
    wind: string;            // 风位
    isHost: boolean;         // 是否房主
    isBot: boolean;          // 是否AI
    hand: Tile[];            // 手牌数组
    melds: Meld[];           // 副露数组
    discards: Tile[];        // 弃牌数组
    flowers: Tile[];         // 花牌数组
    score: number;           // 当前分数
    isTing: boolean;         // 是否听牌
    isQiao: boolean;         // 是否敲牌
    offline: boolean;        // 是否离线
    offlineTime: Date | null;// 离线时间
    aiTakeover: boolean;     // AI接管状态
    sankouCounts: number[];  // 包三口计数器 [4]
}
```

### 2.2 字段详解

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| id | string | - | 真人玩家为 Socket ID，AI 为 'ai_时间戳_座位号' |
| username | string | - | 玩家显示名称 |
| avatar | string | '👤' | 头像 emoji |
| voice | string | 'female01' | 语音类型: 'female01', 'female02', 'male', 'male02' |
| socket | Socket/null | - | 真人玩家有 Socket，AI 为 null |
| ready | boolean | false | 是否准备好开始游戏 |
| seatIndex | number | 0-3 | 座位索引，决定风位 |
| wind | string | - | 根据座位和庄家动态计算 |
| isHost | boolean | false | 是否为房间创建者 |
| isBot | boolean | false | 是否为 AI 玩家 |
| hand | Tile[] | [] | 当前手牌 |
| melds | Meld[] | [] | 副露的顺子/刻子/杠子 |
| discards | Tile[] | [] | 已打出的牌 |
| flowers | Tile[] | [] | 摸到的花牌 |
| score | number | 0 | 当前局得分 |
| isTing | boolean | false | 是否听牌状态 |
| isQiao | boolean | false | 是否已敲牌 |
| offline | boolean | false | 是否离线 |
| offlineTime | Date/null | null | 离线时间戳 |
| aiTakeover | boolean | false | 是否被 AI 接管 |
| sankouCounts | number[4] | [0,0,0,0] | 与其他玩家的三口计数 |

### 2.3 风位计算

```javascript
// 风位根据庄家位置动态计算
// 庄家为东，顺时针确定其他风牌
player.wind = WINDS[(player.seatIndex - dealerIndex + 4) % 4];
```

## 三、副露数据结构

### 3.1 副露对象 (Meld)

```typescript
interface Meld {
    type: 'peng' | 'gang' | 'chi';
    tiles: Tile[];
    from: number;
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| type | string | 副露类型: 'peng'(碰), 'gang'(杠), 'chi'(吃) |
| tiles | Tile[] | 组成副露的牌（碰3张，杠4张，吃3张） |
| from | number | 来源玩家索引（暗杠为自己） |

### 3.2 副露类型详解

| 类型 | 牌数 | 来源 | from 值 | 说明 |
|------|------|------|---------|------|
| peng | 3 | 他人打出 | 出牌者索引 | 碰牌 |
| gang | 4 | 他人打出 | 出牌者索引 | 明杠 |
| gang | 4 | 自己手牌 | 自己索引 | 暗杠 |
| gang | 4 | 自己摸牌+已有碰 | 自己索引 | 加杠 |
| chi | 3 | 上家打出 | 上家索引 | 吃牌 |

## 四、游戏状态数据结构

### 4.1 游戏状态对象 (GameState)

```typescript
interface GameState {
    deck: Tile[];                    // 牌墙
    dealerIndex: number;             // 庄家索引
    currentPlayerIndex: number;      // 当前玩家索引
    turnPhase: 'draw' | 'discard' | 'action';  // 回合阶段
    lastDiscard: Tile | null;        // 最后打出的牌
    lastDiscardPlayer: number;       // 最后出牌玩家索引
    pendingActions: PendingAction[]; // 待处理动作
    actionTimeout: Timeout | null;   // 动作超时计时器
    discardTimeout: Timeout | null;  // 出牌超时计时器
    lastDrawnTile: Tile | null;      // 最后摸的牌
    roundNumber: number;             // 局数
    gameOver: boolean;               // 游戏是否结束
    waitingForQiao: boolean;         // 等待敲牌确认
    huangFanRound: boolean;          // 是否荒番局
    huangFanCount: number;           // 荒番数
    cangyingTile: Tile | null;       // 苍蝇牌
    gangShangPao: boolean;           // 杠上炮标记
    pendingZimo: PendingZimo | null; // 待处理自摸
    pendingAnGang: PendingAnGang | null;   // 待处理暗杠
    pendingJiaGang: PendingJiaGang | null; // 待处理加杠
}
```

### 4.2 回合阶段 (turnPhase)

| 阶段 | 说明 |
|------|------|
| draw | 摸牌阶段，当前玩家需要摸牌 |
| discard | 出牌阶段，当前玩家需要出牌 |
| action | 动作阶段，等待其他玩家响应（碰/杠/吃/胡） |

### 4.3 待处理动作 (PendingAction)

```typescript
interface PendingAction {
    playerIndex: number;     // 玩家索引
    playerId: string;        // 玩家ID
    actions: string[];       // 可执行动作 ['hu', 'gang', 'peng', 'chi']
    tile: Tile;              // 触发动作的牌
    chiOptions?: ChiOption[];// 吃牌选项（如果有吃）
    resolved?: boolean;      // 是否已处理
    action?: string;         // 选择的动作
    selectedChiIndex?: number;// 选择的吃牌选项索引
}
```

### 4.4 待处理自摸 (PendingZimo)

```typescript
interface PendingZimo {
    playerId: string;
    playerIndex: number;
    tile: Tile;
}
```

### 4.5 待处理暗杠 (PendingAnGang)

```typescript
interface PendingAnGang {
    playerId: string;
    playerIndex: number;
    anGangOptions: AnGangOption[];
}

interface AnGangOption {
    tile: Tile;      // 代表牌
    tiles: Tile[];   // 4张相同的牌
}
```

### 4.6 待处理加杠 (PendingJiaGang)

```typescript
interface PendingJiaGang {
    playerId: string;
    playerIndex: number;
    jiaGangOptions: JiaGangOption[];
}

interface JiaGangOption {
    meldIndex: number;  // 副露索引
    tile: Tile;         // 要加杠的牌
}
```

## 五、房间数据结构

### 5.1 房间对象 (MahjongRoom)

```typescript
class MahjongRoom {
    // 基本信息
    code: string;                    // 房间号 (6位)
    hostId: string;                  // 房主ID
    players: Player[];               // 玩家数组 (最多4人)
    viewers: Viewer[];               // 观战者数组
    gameState: GameState | null;     // 游戏状态
    gameRunning: boolean;            // 游戏是否进行中
    createdAt: Date;                 // 创建时间
    
    // 计分系统
    totalRounds: number;             // 总局数 (默认10)
    currentRound: number;            // 当前局数
    matchScores: number[];           // 累计积分 [4]
    roundHistory: RoundResult[];     // 历史记录
    matchStarted: boolean;           // 比赛是否开始
    lastWinnerIndex: number;         // 上局赢家
    
    // 荒番系统
    huangFanCount: number;           // 荒番计数器
    isHuangFanRound: boolean;        // 当前是否荒番局
    
    // 暂停功能
    isPaused: boolean;
    pausePlayer: Player | null;
    pauseStartTime: Date | null;
    pauseCountdown: Interval | null;
    
    // 解散投票
    dissolveRequest: DissolveRequest | null;
    dissolveVotes: Record<string, boolean>;
    
    // 测试房间
    isTestRoom: boolean;
    customDeckOrder: Tile[] | null;
    
    // 自动解散
    autoDissolveTimer: Timeout | null;
    
    // 下一局倒计时
    nextRoundCountdown: number;
    nextRoundTimer: Interval | null;
}
```

## 七、吃牌选项数据结构

### 7.1 吃牌选项 (ChiOption)

```typescript
interface ChiOption {
    type: 'chi';
    tiles: Tile[];        // 组成顺子的3张牌
```

### 7.2 吃牌模式

| 模式 | 说明 | 示例 |
|------|------|------|
| 边张 | 吃边张成顺 | 手牌有1、2万，吃3万成123万 |
| 嵌张 | 吃嵌张成顺 | 手牌有1、3万，吃2万成123万 |
| 边张 | 吃边张成顺 | 手牌有8、9万，吃7万成789万 |
