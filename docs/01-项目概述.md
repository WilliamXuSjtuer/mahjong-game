# 上海敲麻多人联机版 - 项目概述

## 一、项目简介

这是一个基于 Node.js + Socket.IO 的上海敲麻麻将多人联机游戏，支持创建房间邀请好友一起玩。项目采用前后端分离架构，后端提供 WebSocket 实时通信服务，前端通过 Socket.IO 客户端与服务器交互。

## 二、技术栈

| 技术 | 版本 | 用途 |
|------|------|------|
| Node.js | - | 后端运行环境 |
| Express | 4.18.2 | Web 服务器框架 |
| Socket.IO | 4.7.2 | WebSocket 实时通信 |
| compression | 1.8.1 | HTTP 响应压缩 |

## 五、游戏规则概述

### 5.1 基本规则
- **玩家数量**: 4人（真人或AI混合）
- **牌数**: 144张（万条筒各36张 + 风牌16张 + 箭牌12张 + 花牌8张）
- **发牌**: 庄家14张，其他玩家13张
- **胡牌条件**: 4组顺子/刻子 + 1对将（或七对子）

### 5.2 上海敲麻特色规则
1. **敲牌**: 听牌后必须敲牌才能胡牌，敲牌后只能打刚摸的牌
2. **花牌**: 中发白和花牌算花牌，摸到自动补花
3. **包三口**: 同一玩家吃/碰/杠同一玩家3次，需包赔
4. **荒番**: 流局后下一局分数翻倍
5. **杠上炮**: 杠后打出的牌点炮，分数×3
6. **苍蝇**: 胡牌后翻一张牌作为苍蝇，有牌型可额外得分

### 5.3 计分规则
```
分数 = (花数 × 2^番数) + 苍蝇分
荒番翻倍 = 分数 × 2^荒番数
封顶 = 50分
```

```bash

```

- Heroku (Procfile)

## 八、性能优化

### 8.1 Socket.IO 配置
```javascript
{
    pingTimeout: 30000,        // 30秒无响应视为断开
    pingInterval: 10000,       // 10秒心跳间隔
    perMessageDeflate: {       // 消息压缩
        threshold: 512         // 超过512字节才压缩
    }
}
```

### 8.2 状态广播优化
- 轻量级状态更新（仅发送变化数据）
- 广播节流（100ms内只发送一次）
- 按需发送完整状态
