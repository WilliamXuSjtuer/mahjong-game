# 包三口规则实现总结

## 实现概述

已成功为上海麻将游戏添加"包三口"规则，包括：
- 三口计数系统
- 特殊赔付逻辑
- 结算界面显示
- 规则说明文档

## 修改文件

### 1. server.js（服务器端）

#### 玩家数据结构修改
- 在 `addPlayer()` 和 `addAIPlayer()` 函数中添加 `sankouCounts: [0, 0, 0, 0]` 属性
- 用于记录与其他四位玩家的吃碰明杠次数

#### 吃碰杠逻辑修改
- **碰牌** (`action.action === 'peng'`): 碰一次牌，三口计数 +1
- **明杠** (`action.action === 'gang'`): 明杠一次，三口计数 +1
- **吃牌** (`action.action === 'chi'`): 吃一次牌，三口计数 +1
- **暗杠/加杠**: 不计入口数（因为是摸牌，不是吃碰杠）

#### 计分逻辑修改
修改 `calculateScore()` 函数：
- **自摸包三口**: 若胡牌者与某玩家有三口关系（≥3 口），该玩家单独赔付 5 份
- **点炮包三口**: 若胡牌者与放炮者有三口关系（≥3 口），放炮者赔付 2 份
- 返回 `sankouInfo` 对象，包含包三口类型、玩家、口数等信息

## 核心逻辑

### 三口计数
```javascript
// 玩家 A 碰/吃/明杠 玩家 B 的牌
player.sankouCounts[discardPlayerIndex]++;
```

### 包三口检测
```javascript
const checkSankou = (playerIndex, targetIndex) => {
    return this.players[playerIndex].sankouCounts[targetIndex] >= 3;
};
```

### 赔付计算
- **正常自摸**: 三家各付 1 份（共 3 份）
- **自摸包三口**: 包三口者单独付 5 份
- **正常点炮**: 放炮者付 1 份
- **点炮包三口**: 放炮者付 2 份

## 使用示例

### 场景 1：自摸包三口
1. 玩家 A 碰了玩家 B 的牌 3 次
2. 玩家 A 自摸胡牌
3. 结果：玩家 B 单独赔付 5 份给玩家 A

### 场景 2：点炮包三口
1. 玩家 A 吃了玩家 B 的牌 3 次
2. 玩家 B 打出的牌被玩家 A 胡牌
3. 结果：玩家 B 赔付 2 份给玩家 A

## 注意事项

1. **单向关系**: 包三口关系是单向的（A 吃 B 的牌，只计 A 对 B 的口数）
2. **暗杠除外**: 暗杠不计入口数（因为是摸牌）
3. **自摸检查**: 自摸时检查胡牌者对所有其他玩家的口数
4. **点炮检查**: 点炮时只检查胡牌者对放炮者的口数
5. **优先级**: 包三口赔付规则覆盖正常赔付规则
